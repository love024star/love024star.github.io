这两年试过不少知识管理工具，踩了很多坑，慢慢摸索出了一套适合自己的方案。分享出来，希望对你有参考价值。

## 1.0 时代：文件夹 + Word

最早用的是文件夹，按年份建目录，每个项目一个 Word 文档。优点是简单，缺点是：
- 搜索全靠记忆文件名
- 不同设备同步困难
- 内容碎片化，难以关联

## 2.0 时代：Notion 全家桶

Notion 确实很好用，数据库、页面嵌套、模板……用了大半年。但有几个痛点：
- 网络不稳定时体验差
- 导出格式不够灵活
- 团队协作场景有限

## 3.0 时代：GitHub 知识库 + 本地工具

现在的方案：

| 场景 | 工具 |
|------|------|
| 日常笔记 | Obsidian |
| 文档分享 | GitHub 私有仓库 |
| 博客文章 | GitHub Pages + Jekyll |
| 代码片段 | GitHub Gist |
| 团队协作 | 腾讯文档 |

核心思路：**文本文件是最终格式**，不管用什么工具编辑，最终都落地为 Markdown 文件，存在 GitHub 上。

## 最重要的心得

> 工具不重要，习惯才重要。
> 不要花太多时间折腾工具，把时间花在内容上。

找到适合自己的节奏，坚持记录，才是知识管理的本质。
